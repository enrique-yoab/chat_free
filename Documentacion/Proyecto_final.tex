\section{Conclusiónes}

El desarrollo de esta práctica resultó un verdadero reto, principalmente por la necesidad de establecer conexiones P2P y Cliente-Servidor para gestionar distintos algoritmos de sincronización de manera simultánea. Al final, se demostró que estos algoritmos pueden integrarse exitosamente en múltiples procesos, adaptándose a las necesidades de diferentes arquitecturas de red.

Este proyecto me permitió observar de primera mano cómo operan las aplicaciones a bajo nivel. Demostró la importancia vital de mantener un control absoluto sobre los procesos creados (\texttt{forks} e hilos), garantizando su correcta sincronización para evitar problemas lógicos, de concurrencia o de latencia.

Se logró implementar una comunicación efectiva mediante sockets TCP y UDP; sin embargo, esto representa un desafío constante para el programador. Al ser la base sobre la cual opera toda aplicación de red, su complejidad exige un diseño sumamente estructurado. Quedó claro que es imposible comprender, modificar o escalar un sistema distribuido si no se cuenta con un control de ejecución riguroso y una documentación estricta sobre el ciclo de vida de cada proceso.
